# プロジェクト指示書：自宅IoT通知PWA「通知君(仮)」開発

## 1. プロジェクト概要

本プロジェクトは、自宅のIoTデバイスや各種サービスからの通知を一元管理するためのAndroid専用PWA（Progressive Web App）を開発することを目的とします。特に、スマートロックSesameの解錠通知にボタンを付加し、通知上から直接施錠できる機能を核とします。

## 2. アプリケーション要件

### 2.1. 機能要件

* **Sesame スマートロック連携**:
    * Sesameの解錠状態を検知した際、PWAを通じてAndroid端末にプッシュ通知を送信すること。
    * 通知には「施錠」ボタンを表示し、ユーザーがこれをタップすると施錠コマンドが実行されること。
    * 通知メッセージには、デバイス名とバッテリー残量を表示すること。
* **汎用通知機能**:
    * Googleカレンダー、買い物リストなど、Webhook経由で送信される様々な自宅関連の通知を受け取り、PWAとして表示すること。
* **PWAインストール**:
    * Android端末のホーム画面に追加することで、ネイティブアプリのように利用できること。

### 2.2. 非機能要件

* **対象OS**: Android専用とする。iOSは通知ボタンがサポートされていないため対象外とする。
* **ホスティング**: 自宅サーバー上でバックエンドとPWAをホスティングすること。
* **セキュリティ**:
    * サーバー側の設定により、特定のAndroid端末からのみPWAとしてインストールできるように制限すること。
    * PWAからの施錠コマンド送信は、セキュアな方法（例：API署名）で行うこと。
* **通知の仕組み**: Web Push APIとNotifications APIを利用して、ブラウザが閉じた状態でも通知を受信できるようにすること。

---

## 3. 技術スタック

* **バックエンド/サーバー**: Python (Flask, FastAPIなど)
* **PWA**: HTML, CSS, JavaScript
* **プッシュ通知**: Web Push API, Notifications API, Service Worker
* **認証・認可**: ベーシック認証、またはIPアドレス制限と、端末ごとのユニークなIDを利用したホワイトリスト方式

---

## 4. 詳細仕様

### 4.1. バックエンド（通知受信サーバー）

#### 4.1.1. Webhookエンドポイント

以下の2つのWebhookエンドポイントを実装する。

* **`POST /webhook/sesame`**: Sesameの状態変更通知用
    * **受け取るデータ**: `device_id`, `state` ("locked" or "unlocked"), `battery`など、必要な情報をJSON形式で受け取る。
    * **処理**: `state`が"unlocked"の場合、PWAにプッシュ通知を送信する。
* **`POST /webhook/general`**: 汎用通知用
    * **受け取るデータ**: `title`, `body`, `type`など、汎用的な通知内容をJSON形式で受け取る。

#### 4.1.2. PWAへのプッシュ通知

* 通知のペイロードに、通知内容だけでなく、アクションボタンの情報を追加する。
* Sesame通知の場合、ペイロードに`device_id`や`secret_key`など、施錠コマンドの実行に必要な情報を暗号化して含める。

### 4.2. PWAクライアント

#### 4.2.1. Manifestファイル (`manifest.json`)

* PWAの名称 (`"name"`, `"short_name"`) を「通知君」に設定する。
* `"icons"`、`"start_url"`、`"display"`などを適切に定義し、ホーム画面への追加を可能にする。

#### 4.2.2. Service Worker (`service-worker.js`)

* **`push`イベントリスナー**: サーバーから送られてくるプッシュメッセージを待ち受ける。
* **`notificationclick`イベントリスナー**: ユーザーが通知上のボタンをタップした際に発火する。
    * `event.action`プロパティで、どのボタンが押されたかを判定する。
    * Sesame施錠ボタンが押された場合、`lock_sesame`関数を実行し、施錠コマンドを送信する。

#### 4.2.3. 通知の表示

* **`self.registration.showNotification()`**メソッドを使用する。
* `options`オブジェクトに`"actions"`プロパティを設定し、ボタンを定義する。
    * 例：`actions: [{ action: 'lock', title: '施錠' }]`
    * `action`には、`notificationclick`イベントで判定するためのユニークな識別子を設定する。

### 4.3. Sesame施錠コマンドの詳細

* **署名の生成**:
    * 施錠コマンドの送信には、AES-CMAC署名が必要です。
    * 既存の`generate_sesame_sign`関数（Python）のロジックを参考に、PWA側で署名を生成するためのJavaScript関数を作成する。
    * この関数は、デバイスのシークレットキーと現在のUNIXタイムスタンプを基に、3バイトのメッセージを生成し、CMAC署名を計算するものです。
* **コマンド送信**:
    * `POST`リクエストをSesame APIの施錠エンドポイント (`/api/sesame2/{device_id}/cmd`) に送信する。
    * ペイロードには、`cmd: 82`、`history`、そして生成した`sign`を含める。

---

## 5. セキュリティ要件の詳細

* **PWAインストール制限**:
    * サーバーに、許可された端末のユニークなID（例：UUID、またはブラウザのフィンガープリント）のホワイトリストを保持する。
    * PWAのインストールプロセス（`beforeinstallprompt`イベント）や、Service Workerの登録時に、端末IDをサーバーに送信して認証を行う。
    * ホワイトリストにない端末からのアクセスはブロックし、PWAとしてインストールできないようにする。

---

## 6. 開発フロー

1.  バックエンドサーバーを構築し、Web Pushに対応させる。
2.  PWAの`manifest.json`と`service-worker.js`を作成する。
3.  SesameのWebhookハンドラーを実装し、テストする。
4.  PWAの通知表示とボタン操作を実装し、テストする。
5.  セキュリティ機能（端末制限）を実装する。
6.  その他の通知（カレンダー、買い物リスト）用の汎用Webhookエンドポイントと通知機能を実装する。
